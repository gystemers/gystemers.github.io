2026年1月

2026-01-05 Mon

研究室のPCのEmacs設定ファイルを家のものと同期した所，初起動のvtermがコンパイルできなかったり，Emacs からフォーカスが外れたときに日本語ソースを com.apple.keylayout.ABC から com.apple.keylayout.DVORAK-QWERTYCMD に動してくれていた仕組みが動かなくなっちゃったりで，色々と試行錯誤．
vtermはインストールされているはずの「CMakeが見つからない」というエラーが出ていたが，これは MacPortsのCMake と HomebrewのCMake の両方をインストールしてしまっていて，参照先が喧嘩していたことが原因だったようだ．

sudo port uninstall cmake and installed

で MacPorts側のCMakeを削除して，Emacs内でも

which cmake

の参照先をRefすることで事なきを得た．
Emacs からフォーカスが外れても macism でMacOSの入力ソースが ABC から Dvorak+QWERTY⌘ に変わらない問題は，使っていた「add-hook focus-in-hook」などのコマンドがオワコンとなっていたからのようで，M365 copilot (GPT 5.2) の助けを借りながら，新しい hook コマンドで書き直すことでなんとか決着．
やはりこういう時に Lisp言語 などの本質を全く自分が理解していないことを痛感する．
……のだが，物理と違ってアプリはあくまで「道具」なので，そこはAIに任せて自分の無駄な時間を割くのはやめよう……と，そっとプログラム本を閉じたのだった．

2026-01-04 Sun
